Virtual DOM
  Виртуальный DOM (VDOM) - это концепция программирования, в которой идеальное или
  «виртуальное» представление пользовательского интерфейса хранится в памяти и
  синхронизируется с «настоящим» DOM при помощи библиотеки, такой как ReactDOM.
  Этот процесс называется согласованием (reconcilation).

  Вместо того, чтобы взаимодействовать с DOM напрямую, мы работаем с его легковесной
  копией (легким JavaScript-объектом). Мы можем вносить изменения в копию, исходя из
  наших потребностей, а после этого применять изменения к реальному DOM.
  При этом происходит сравнение DOM-дерева с его виртуальной копией, определяется
  разница и запускается перерисовка того, что было изменено.

Light DOM
  Обычное DOM дерево, структуру которого можно увидеть в HTML разметке.

Shadow DOM
  Теневой DOM - это браузерная технология, предназначенная в основном для определения
  области видимости переменных и CSS в веб-компонентах.

  Проблемы и  возможные коллизии в построении компонентов с использование HTML/JS:
    - стили документа могут повлиять на элементы в компоненте;
    - JS может каким-либо образом нежелательно модифицировать элементы компонента;
    - ID элементов компонента могут совпадать с ID на странице.

  Shadow DOM инкапсулирует поддерево DOM и стили, исключая возникновение подобных
  проблем.

  Можно рассмотреть это на примере тега <video> - shadow dom скрывает структуру этого
  элемента, инкапсулируя содержимое и исключая вышеописанные проблемы.

Использование shadow DOM в react: https://www.npmjs.com/package/react-shadow
