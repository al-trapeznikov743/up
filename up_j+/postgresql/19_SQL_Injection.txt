SQL Injection

SQL-инъекция - по сути своей это попытка изменить запрос к базе данных. Осуществляется
это дело через пользовательский интерфейс (к примеру форму), GET-запрос (в поисковой
строке браузера) и т.д.

Правила безопасности в случае с SQL

  1) Плейсхолдеры - подстановка данных
    Любые данные должны попадать в запрос не напрямую, а через некоего представителя,
    подстановочное выражение.

    Пример:
      SELECT * FROM table WHERE id > ? LIMIT ?

    Форматируем данные непосредственно перед вставкой в выражение и исполнением запроса -
    таким образом мы будем уверены в том, что данные форматируются корректно и далее они
    попадут строго по назначению - в БД.

  2) Белые списки
    Идентификаторы и ключевые слова подставляем только из белого списка, прописанного в
    нашем коде.

  Так же хотелось бы сказать о таких валидирующих методах защиты на уровне пользовательского
  интерфейса (формы, поля) как:
    - маски (к примеру с использованием регулярных выражений);
    - удаление лишних пробелов;
    - экранирование некоторых символов.
