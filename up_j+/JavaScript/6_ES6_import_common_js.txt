CommonJS (CJS)
  - это добровольная рабочая группа, которая проектирует, прототипирует и стандартизирует
  различные JavaScript API.

  На сегодняшний день они ратифицировали стандарты для модулей и пакетов.

  CommonJS определяют простой API для написания модулей, которые могут быть использованы в
  браузере с помощью тега <script>, как с синхронной, так и с асинхронной загрузкой.

  Главное воплощение этого стандарта — модульная система Node.js (в Node.js есть несколько фич,
  выходящих за рамки CJS).
  
  Характеристики стандарта:
    - компактный синтаксис;
    - предназначен для синхронной загрузки;
    - преимущественно используется на стороне сервера.

  В структурном плане, CommonJS-модуль представляет собой готовый к переиспользованию фрагмент
  JavaScript-кода, который экспортирует специальные объекты, доступные для использования в
  любом зависимом коде.

  Обычно описываются две главные идеи:
    - объект exports, содержащий то, что модуль хочет сделать доступным для других частей
      системы;
    - функция require, которая используется одними модулями для импорта объекта exports из
      других.

ES6 Module Syntax
  Модули ECMAScript 6 очень похожи на модули Node.js. Модуль - это просто файл с JavaScript
  кодом внутри.

  Вы можете писать код в отдельном файле и экспортировать нужные вещи с помощью "export" в
  виде модулей. Другие файлы могут импортировать (import) модули того файла для использования.

  В дополнение к декларативному синтаксису для работы с модулями, в стандарте ECMAScript 6
  также присутствует программный API. Он позволяет делать две вещи:
    - программно работать с модулями и скриптами;
    - настраивать загрузку модулей.
